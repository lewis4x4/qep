# B1.3 / QEP-47 — AI Briefing Depth Check

Date: 2026-05-21
Source: `docs/operations/IRON_FLOOR_AUDIT_2026-05-17.md` §3.3
Route: `/floor` for `iron_advisor`

## Verdict

**Blocked on operational signoff, not blocked on code.**

The current advisor briefing is no longer only static copy. `AdvisorBriefingBanner` combines live Sales Companion briefing data, live priority actions, QRM advisor pipeline stats, closing-soon count, and explicit degraded/empty states. That is enough for an operational review pass.

The remaining question is product judgment: Rylee's phrase “AI briefing for the day” may require named deals, named blockers, and specific next-move recommendations in the top banner itself. The current banner summarizes counts/value/priority pressure, while named work lives in adjacent widgets/cards.

## Code evidence

- `apps/web/src/features/floor/components/AdvisorBriefingBanner.tsx`
  - Loads `useTodayFeed()` for briefing, stats, priority actions, pipeline, and errors.
  - Loads `fetchAdvisorPipelineStats()` for QRM active pipeline, decision-stage count, and total value.
  - Builds summary parts from:
    - QRM active pipeline dollars
    - Sales Companion pipeline dollars
    - active deal count
    - decision-stage count
    - deals closing this week
    - distinct priority actions today
  - Shows explicit degraded-state copy when daily briefing, Sales pipeline, or QRM signals fail.

- `apps/web/src/features/sales/components/AiBriefingCard.tsx`
  - Renders time-of-day greeting and the summary assembled by the floor banner.
  - Preserves fallback behavior for callers without richer `summaryParts`.

- `apps/web/src/features/floor/hooks/useFloorNarrative.ts` and `apps/web/src/features/floor/lib/static-narrative.ts`
  - Provide fallback role-home narrative.
  - The static advisor fallback is intentionally generic: “today's selling motion is ordered by the next action most likely to move a deal.”

## What is sufficient today

The briefing can truthfully communicate:

- pipeline value
- active deal count
- decision-stage pressure
- closing-soon count
- priority-action count
- source health/degraded states

This satisfies a lightweight “start of day orientation” standard.

## What remains for Rylee/Brian signoff

Open `/floor` against real production/staging advisor data and decide whether the top banner must name specific records, for example:

- “Call Acme about the RT-135 quote before Friday.”
- “2 decision-stage deals need manager approval.”
- “Beacon Ridge has a stale follow-up and $184K at risk.”

If that level of specificity is required, the follow-up build should extend `AdvisorBriefingBanner` / `useTodayFeed` summary assembly, or add an advisor-scoped edge function similar to `owner-morning-brief` that emits named, ranked next moves.

## Recommended roadmap status

Keep B1.3 **blocked/manual-pending** until Rylee/Brian review a real advisor dataset. No code change is required before that review.
